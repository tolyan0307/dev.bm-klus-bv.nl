# SEO WORKFLOW — Пошаговая инструкция взаимодействия

> Этот файл — твоя рабочая инструкция. Открывай перед каждой SEO-сессией.

---

## Принцип работы

- Работаем **постранично** (одна страница за сессию).
- Внутри страницы — **посекционно** (можно принять/отклонить каждую секцию отдельно).
- Каждая страница проходит **3 шага**: Анализ → Правки → QA.
- Ты принимаешь **стратегические решения** (structure, intent, business facts).
- Cursor проверяет **качество NL текста, SEO, и каннибализацию**.

## Перед началом работы над страницей

### Что делаешь ты (1 минута):
1. Определи страницу из очереди
2. Если есть новые данные из Semrush/Ahrefs — приготовь (ключевики, позиции)
3. Всё. Остальное делает Cursor.

### Что делает Cursor автоматически:
- Читает brief (или создаёт из шаблона)
- Читает текущий контент страницы
- Читает sibling pages из brief
- Проверяет SERP (если нужно)
- Следует GLOBAL_SEO_CONTENT_RULES.md

## Шаг 1 — Анализ + Outline

### Твой промпт:

```
Шаг 1 для /[slug]/

Прочитай:
- seo-system/GLOBAL_SEO_CONTENT_RULES.md
- seo-system/briefs/[slug].yaml
- текущий файл страницы и контент

Сделай анализ + outline по CURSOR_TASK_TEMPLATE.md Step 1.
```

Если brief ещё не создан:

```
Создай brief для /[slug]/ по шаблону seo-system/SEO_BRIEF_TEMPLATE.yaml.
Сохрани в seo-system/briefs/[slug].yaml.
Потом выполни Шаг 1 (анализ + outline).
```

Если есть данные из Semrush/Ahrefs:

```
Шаг 1 для /[slug]/

Данные из Semrush:
- Primary: [keyword] (volume: X, KD: Y)
- Secondary: [keyword1], [keyword2], ...
- Текущая позиция: Z

Прочитай seo-system/ правила и brief, сделай анализ + outline.
```

### Что ты проверяешь в ответе:
- [ ] Outline логичный? (структура H1/H2/H3 имеет смысл?)
- [ ] Есть CTA?
- [ ] Страница не пытается покрыть "всё обо всём"?
- [ ] Бизнес-факты верны? (цены, сроки, сертификаты)
- [ ] Нет пересечения с другими страницами?

### Твой ответ:
- Если всё ок: **"Outline принят. Делай Шаг 2."**
- Если нужны правки: **"Поправь: [что именно]. Потом Шаг 2."**
- Если нужно обсудить: задай вопрос

---

## Шаг 2 — Правки контента (по секциям)

### Твой промпт:

```
Шаг 2 для /[slug]/

Outline принят. Внеси правки по секциям по CURSOR_TASK_TEMPLATE.md Step 2 (improve).
```

Или для новой страницы:

```
Шаг 2 для /[slug]/

Outline принят. Напиши контент по секциям по CURSOR_TASK_TEMPLATE.md Step 2 (create).
```

### Что ты проверяешь:
- [ ] Правки точечные, не переписан весь файл?
- [ ] Бизнес-факты не изменились без флага?
- [ ] Если есть `[CLAIM_NEEDS_CONFIRMATION]` — подтверди или отклони
- [ ] Структура компонентов не сломана?

### Твой ответ:
- Если всё ок: **"Принято. Делай Шаг 3 (QA)."**
- Если нужны правки секции: **"Секция [название] — переделай: [что не так]"**

---

## Шаг 3 — QA + Финализация

### Твой промпт:

```
Шаг 3 для /[slug]/

Проведи QA по CURSOR_TASK_TEMPLATE.md Step 3.
```

### Что ты проверяешь в ответе:
- [ ] Все 6 оценок ≥ 70? (SEO, NL quality, claims safety, conversion, cannibalization, on-page)
- [ ] Нет FAIL?
- [ ] Если PASS WITH FIXES — правки уже применены?
- [ ] Meta title и description в рамках лимитов?

### Твой ответ:
- Если PASS: **"Принято. Следующая страница."**
- Если PASS WITH FIXES: **"Применяй исправления."** → проверяешь → принимаешь
- Если FAIL: **"Что именно переделать?"** → возврат к Шагу 2 для конкретных секций

## Особые сценарии

### Только QA (без правок)
Если хочешь просто проверить существующую страницу без изменений:

```
Только QA для /[slug]/

Прочитай seo-system/ правила, brief и текущий контент.
Проведи QA аудит по CURSOR_TASK_TEMPLATE.md Step 3.
Не вноси изменений, только отчёт.
```

### Batch QA для location pages
Для 21 location page по одной делать неэффективно:

```
Batch QA для location pages.

Прочитай lib/content/gevelisolatie-locations.ts и app/gevelisolatie/[location]/page.tsx.
Прочитай seo-system/GLOBAL_SEO_CONTENT_RULES.md.

Проверь:
1) Уникальность контента между городами (не copy-paste?)
2) Локальные факты корректны? (bouwperiode, besparing, afstanden)
3) Цены консистентны?
4) FAQ не дублируются?
5) Meta titles/descriptions уникальны и в лимитах?

Дай сводную таблицу: город → оценка → проблемы.
```

## Типичная сессия (20–30 минут на страницу)

```
Минута 0:   "Шаг 1 для /gevelisolatie/"
Минута 3:   Читаешь outline → "Outline принят. Делай Шаг 2."
Минута 5:   Cursor вносит правки по секциям
Минута 15:  Проверяешь → "Принято. Делай Шаг 3."
Минута 17:  QA-отчёт
Минута 20:  "PASS WITH FIXES — применяй." или "Принято. Следующая страница."
```

За один вечер реально обработать 3–5 страниц.

## Чего НЕ нужно делать

- Не загружай весь Excel/CSV из Semrush в чат — только ключевики для конкретной страницы
- Не проси "напиши всю страницу сразу" — всегда 3 шага
- Не пропускай QA (Шаг 3) — это единственная проверка качества NL
- Не правь файлы seo-system/ вручную без причины — они настроены под проект
- Не меняй порядок волн без стратегической причины — money pages дают ROI первыми
