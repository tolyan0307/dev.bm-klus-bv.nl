# 60 — Decisions & Bans

> Explicit bans and architectural decisions that must not be reversed without discussion.
> "Decided" = tested, shipped, deliberate. Do not "improve" these silently.

---

## Hard bans

| Ban | Reason |
|-----|--------|
| Do not install ESLint | Not in devDependencies — use `npx tsc --noEmit` |
| Do not use npm or yarn | pnpm is the package manager |
| Do not use Next.js `<Image>` component | Custom `<ResponsiveImage>` system is used (docs/IMAGE-PIPELINE.md) |
| Do not write hex colors in className | Use design tokens except in hero gradient `style={}` |
| Do not import `buildSrcSet` / `getFallbackSrc` / `image-manifest.json` in `"use client"` components | Causes ~47KB manifest leak into client bundle |
| Do not add Google Analytics outside GTM | Single GTM integration point exists |
| Do not create pages without `data/sitemap-plan.ts` entry | Breaks sitemap generation |
| Do not use emoji as UI icons | lucide-react only |
| Do not output public content in Russian or English | Dutch (nl-NL) only on public pages |
| Do not fabricate business facts | KVK, certifications, prices, project counts — always flag `[CLAIM_NEEDS_CONFIRMATION]` |
| Do not add mid-page CTA blocks | Hero + StickyCTABar pattern is sufficient |
| Do not use `"use client"` at page level | Exception: `/contact/page.tsx` (accepted, do not refactor) |
| Do not hand-edit `data/image-manifest.json` | Generated by `scripts/generate-variants.mjs` |
| Do not source images from `public/images/` | Source always from `source-images/` |

## Architectural decisions

| Decision | Rationale |
|----------|-----------|
| `output: 'export'` static SSG | Apache hosting on Antagonist — no server runtime available |
| All content in `lib/content/*.ts` | No CMS; content is version-controlled and type-safe |
| Trailing slashes on all URLs | Enforced by `next.config.mjs`; Apache rewrite rules match |
| QuoteModal triggered via `#offerte` | Consistent CTA pattern; avoids full-page navigation |
| WhatsApp primary, phone secondary | Business preference — owner confirmed |
| No mid-page CTA blocks | Hero + StickyCTABar is sufficient; removed from all service pages |
| FAQ with native `<details>` on service pages | Server-renderable, zero client JS, accessible |
| `WaaromBmKlusSection` reused across service pages | Single component with `subtitle` prop — maintains consistency |
| Dynamic imports for QuoteModal, StickyCTABar, ReviewsSection | Code-split — not needed on initial render |

## Performance decisions

| Decision | Rationale |
|----------|-----------|
| `/contact/page.tsx` stays `"use client"` | Complex form state — refactor cost > benefit. Accepted. |
| `/gevelisolatie/materialen/page.tsx` must be refactored | Currently the only unnecessary page-level client boundary. Priority action. |
| Vendor runtime chunks: leave alone | Next.js RSC/polyfills are framework fixed costs |
| GTM integration: do not change | Clean, single integration point at `components/gtm-provider.tsx` |
| Tailwind CSS: do not attempt per-route splitting | Not supported in v4 without major custom tooling |
