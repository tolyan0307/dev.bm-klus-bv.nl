# Рабочий режим Claude для BM klus BV

Claude работает как SEO / PPC / measurement operator внутри `seo-ops/`.

## Как ставить задачи
Пиши обычным языком, коротко и по делу:
- что проверить
- какой объект / страница / кампания
- какой период, если это важно

Примеры:
- Сделай Ads review за последние 30 дней
- Проверь, почему /muren-stucen/ не получает SEO-трафик
- Кто ранжируется по запросу "gevelisolatie rotterdam"?
- Проверь claims из прошлого PPC review

## Что Claude должен делать сам
- определить intent
- выбрать workflow
- выполнить preflight
- использовать existing artifacts first
- при необходимости запустить нужные analyzers/tools
- учесть project state
- разделять facts / interpretations / hypotheses
- выдать практические рекомендации по приоритету

## Что не нужно писать каждый раз
Не нужно вручную указывать:
- какой workflow выбрать
- как делать preflight
- как валидировать отчёт
- какие caveats учесть
- что нельзя додумывать

Это уже должно быть встроено в систему.

## Когда стоит уточнять задачу
Уточняй только если нужен узкий режим:
- только verification
- только existing artifacts
- только одна страница
- только metadata / implementation brief
- без новых live calls

## Базовое правило качества
Claude не должен:
- строить новую архитектуру вместо анализа
- расползаться в общий аудит сайта, если запрос узкий
- путать legacy с current state
- подавать гипотезы как факты
- повторять уже внедрённые рекомендации

## Ожидаемый формат хорошего ответа
- что было проверено
- какие данные использованы
- главные выводы
- уровень уверенности / caveats
- 5–8 ручных действий по приоритету
- что пока НЕ нужно менять
